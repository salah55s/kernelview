# Dashboard Guide

## Overview

The KernelView Dashboard is a React + TypeScript single-page application built with Vite, Tailwind CSS, and shadcn/ui components. It provides real-time visibility into cluster health, service dependencies, incidents, and resource optimization.

## Accessing the Dashboard

```bash
# Port-forward to your local machine
kubectl port-forward svc/kernelview-dashboard 8080:8080 -n kernelview

# Open in browser
open http://localhost:8080
```

For production, expose via your Ingress controller with TLS.

## Pages

### 1. Service Map (`/map`)

A **D3.js force-directed graph** that visualizes all discovered services and their dependencies. Each node is a service; each edge represents observed HTTP/gRPC traffic between services.

**Node colors:**
- 🟢 Green: Healthy (error rate < 1%, latency normal)
- 🟡 Yellow: Degraded (error rate 1-5% OR latency elevated)
- 🔴 Red: Critical (error rate > 5% OR active P0/P1 incident)
- 🔵 Blue: No data yet (recently discovered)

**Interactions:**
- Click a node to navigate to the Service Detail page
- Hover to see quick stats (RPS, p99 latency, error rate)
- Nodes with active incidents pulse with an animated glow

### 2. Service Detail (`/services/:name`)

Detailed telemetry for a single service, with four charts:

| Chart | Metric | Window |
|-------|--------|--------|
| Latency | p50, p95, p99 response time | Last 30 minutes |
| Throughput | Requests per second | Last 30 minutes |
| Error Rate | 4xx and 5xx percentage | Last 30 minutes |
| CPU/Memory | Container resource usage vs limits | Last 30 minutes |

Below the charts: a **Trace Table** showing the most recent HTTP/gRPC requests with status, latency, and trace ID. Click any row to inspect the full trace.

### 3. Incidents (`/incidents`)

List of all active and recent incidents. Each incident card shows:

- Severity badge (P0 🔴, P1 🟠, P2 🟡, P3 🔵)
- Incident type code (e.g., `MEM-001`)
- Affected service and namespace
- Time since detection
- Current state (DETECTED → CLASSIFYING → CORRELATING → ...)

Click an incident to see the **Incident Detail** view with:
- Full timeline of events
- AI root cause analysis (Enterprise)
- Remediation actions taken
- Safety check results

### 4. Right-Sizing (`/right-sizing`)

Resource optimization recommendations based on observed usage patterns. Displays a table of services with:

| Column | Description |
|--------|-------------|
| Service | Service name and namespace |
| CPU Request/Limit | Current vs recommended |
| Memory Request/Limit | Current vs recommended |
| Monthly Savings | Estimated cost reduction |
| Action | "Apply" button to patch the deployment (Enterprise) |

Recommendations are generated by comparing the p99 resource usage over 7 days against the declared requests/limits. A service requesting 1 CPU but never exceeding 200m is flagged with a recommended request of 250m (p99 × 1.25 headroom).

## Theming

The dashboard supports **dark mode** (default) and light mode, toggled via the sun/moon button in the header.

- Fonts: Inter (UI), JetBrains Mono (code/metrics)
- Colors: shadcn/ui design system with custom blue/purple accent
- Effects: Glassmorphism on panels, backdrop blur on header, smooth transitions

## Configuration

The dashboard reads its API URL from environment variables:

```env
VITE_API_URL=http://localhost:8080/api/v1
```

In production (Helm deployment), this is set automatically via ConfigMap.
